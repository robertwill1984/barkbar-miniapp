# 现金流与运营支出管理规则 V1.0

版本：V1.0｜日期：2026-07-09

## 1. 独立账簿

系统至少保存六类不可混用的事实：

1. 支付收款事实；
2. 收入确认事实；
3. 待兑付权益事实；
4. 退款事实；
5. 运营支出事实；
6. 现金账户流水与余额。

利润不等于现金；收款不等于收入；采购付款不一定等于当期商品成本。

## 2. 现金流水

每条流水包含：账户、方向、业务类型、金额、发生时间、价值日期、渠道单号、业务单号、对手方、经办人、凭证、状态和冲正关联。

`期末现金余额 = 期初现金余额 + 期间现金流入 - 期间现金流出`

流入：微信支付、其他收款、股东投入（非收入）、借款（非收入）、退款退回等。  
流出：顾客退款、采购、工资、房租、水电、营销、维修、税费、平台费及其他。

账户余额按微信商户、银行、现金等分别计算，再汇总；账户间转账一进一出，不得算收入或支出。

## 3. 收款

- 以支付渠道成功状态为准，支付回调与主动查单使用同一支付单幂等。
- 支付金额、优惠、渠道补贴、手续费分列。
- 未支付、失败、关闭订单不计收款。
- 每日与微信支付账单对账；差异进入待处理，不允许人工改原支付。

## 4. 运营支出

分类：房租、人工、水电、采购、营销、设备、维修、耗材、税费、平台手续费、顾客退款、其他。

每笔支出必填：分类、金额、业务日期、所属月份、收款方、支付账户、支付方式、经办人、状态。凭证图片、发票状态、合同/采购单、备注按规则要求。

状态：`DRAFT → SUBMITTED → APPROVED → PAID → REVERSED`；驳回为 `REJECTED`。已支付记录不得编辑金额，只能冲正后重录。

固定支出支持周期模板，但每月仍生成独立实例并确认支付。

## 5. 商品成本

- 采购付款属于现金流出；采购入库形成库存资产，不立即全部计入当期商品成本。
- 商品交付时按选定成本法结转销售成本。首期推荐移动加权平均法。
- 报损、领用、盘亏分别进入损耗/运营成本，不得伪装为销售成本。
- 待确认：财务报表采用含税还是不含税成本；推荐内部经营看板同时保存税额，默认用实际承担的不含可抵扣税成本。

## 6. 退款

- 退款申请与退款成功分开；现金只在渠道确认成功后减少。
- 顾客退款不重复计作一般运营支出；报表单列，避免既冲收入又计支出造成双重影响。
- 支付手续费是否退回按渠道账单记录。

## 7. 核心报表公式

- 当日实际收款：支付成功时间在当日的成功支付毛额。
- 当日退款：退款成功时间在当日的退款额。
- 当日净收款：实际收款 - 退款。
- 当月确认收入：确认日期在当月的收入确认净额。
- 本月总现金流出：所有已完成流出，不含账户内部转账。
- 本月净现金流：现金流入 - 现金流出。
- 商品毛利：商品确认收入 - 对应销售成本。
- 经营贡献毛利（内部口径）：确认收入 - 商品成本 - 可直接归属变动成本。
- 现金覆盖月数：可动用现金余额 ÷ 最近3个月平均固定现金支出。分母为0时显示“不可计算”。

每个指标必须保存计算版本、时间范围、数据更新时间，并支持下钻。

## 8. 月结与调整

- 每月由财务执行对账、待兑付勾稽、库存结转和支出复核。
- 月结后锁定期间。补录使用“调整期 + 原业务日期”，不得篡改已关账快照。
- 财务调整记录调整前值、调整后值、原因、证据、申请人、审批人和时间。
- 老板可重开期间，但必须二次确认并记录。

## 9. 预算与异常提醒

- 支出预算按月份和分类设置；达到 80% 提醒，达到 100% 警告，超预算需审批。
- 现金余额低于未来 2 个月固定支出、单类支出环比异常、库存采购高于销售速度时预警。
- 预测只作为经营参考，展示假设、样本期和误差，不写入正式流水。

## 10. 数据来源

- 收款/退款：支付记录和微信支付账单。
- 收入：收入确认记录。
- 待兑付：权益负债流水。
- 支出：审批通过且已支付的支出及账户流水。
- 现金：各现金账户流水。
- 成本：采购入库、库存流水、订单交付。

## 11. 待确认

1. 可动用现金是否排除押金、受限资金和股东专项款。
2. 固定支出分类及现金安全线（月数）。
3. 支出审批金额分级。
4. 移动加权平均成本是否满足首期管理要求。
5. 月结日期、补录期限和老板重开权限。
6. 税额、平台手续费在内部经营报表中的展示口径。
