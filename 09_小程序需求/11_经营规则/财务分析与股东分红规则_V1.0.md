# 财务分析与股东分红规则 V1.0

日期：2026-07-10  
性质：内部经营管理口径；法定会计利润、税务申报利润和正式股东分红方案必须由会计/税务/法律专业人员复核。

## 1. 核心结论

后端必须支持财务分析，包括现金流、内部经营利润测算和股东分红依据。

但系统需要明确区分：

1. 现金余额；
2. 可动用现金；
3. 内部经营收入；
4. 内部经营利润；
5. 法定会计利润；
6. 可分配利润；
7. 实际股东分红。

内部经营利润可以帮助老板和股东看经营情况，但不能自动等同于法定可分红金额。

## 2. 三张核心报表

### 2.1 现金流表（经营管理口径）

用于判断店里现金是否安全。

关键指标：

- 期初现金余额；
- 现金流入；
- 现金流出；
- 期末现金余额；
- 可动用现金；
- 受限资金；
- 股东投入；
- 股东借款；
- 现金覆盖月数。

公式：

```text
期末现金余额 = 期初现金余额 + 期间现金流入 - 期间现金流出
可动用现金 = 可提现微信余额 + 可自由使用银行余额 + 门店备用现金
现金覆盖月数 = 可动用现金 ÷ 最近3个月平均固定现金支出
```

股东投入、股东借款、押金、受限资金不得计作营业收入。

### 2.2 利润测算表（内部经营口径）

用于看门店是否真的赚钱。

关键指标：

- 内部经营收入；
- 商品销售成本；
- 会员履约成本；
- 营销/获客成本；
- 人工；
- 房租；
- 水电；
- 维修；
- 系统/平台费用；
- 税费；
- 其他运营支出；
- 内部经营利润。

推荐公式：

```text
内部经营毛利 = 内部经营收入 - 商品销售成本 - 可直接归属变动成本
内部经营利润 = 内部经营收入 - 商品销售成本 - 会员履约成本 - 营销成本 - 运营支出
```

注意：

- 会员卡收款不能一次性全计入收入；
- 未履约权益必须保留为待兑付；
- 商品采购付款不一定等于当期商品成本；
- 退款不能既冲收入又重复计作一般支出；
- 折旧、摊销、税费和人工归类需会计复核。

### 2.3 股东分红依据表

用于给股东看“是否具备分红条件”和“建议分红金额上限”。

系统只提供测算依据和审批记录，不自动决定分红。

建议至少展示：

- 本期内部经营利润；
- 累计内部经营利润；
- 本期净现金流；
- 当前可动用现金；
- 未来3个月固定支出预留；
- 未履约权益余额；
- 待退款/争议/应付款预留；
- 税费及会计调整预留；
- 建议可分红上限；
- 老板确认金额；
- 股东实际分红付款记录。

建议公式：

```text
分红前安全现金 = 可动用现金 - 未来3个月固定支出预留 - 未履约权益风险预留 - 待退款/争议预留 - 税费及会计调整预留
建议可分红上限 = min(内部经营累计可分配利润, 分红前安全现金)
```

如果任一结果小于等于0，系统应提示“不建议分红”。

## 3. 分红前必须拦截的风险

以下情况出现时，系统应提示暂缓分红或要求老板二次确认：

- 当月账期未关账；
- 微信支付、银行、现金账户未完成对账；
- 存在未处理退款、事故争议或大额应付款；
- 可动用现金低于未来3个月固定支出；
- 未履约权益余额过高；
- 季卡/年卡收入仍有大量未摊销；
- 期初余额、股东投入、股东借款未确认；
- 会计/税务口径尚未确认。

## 4. 后端必须保存的数据

### 4.1 已有基础事实

沿用现有数据库设计：

- `payments`：支付收款；
- `revenue_recognitions`：内部收入确认；
- `deferred_entitlement_ledger`：待兑付权益；
- `cash_accounts`：现金账户；
- `cash_ledger`：现金流水；
- `operating_expenses`：运营支出；
- `financial_adjustments`：财务调整。

### 4.2 建议新增或预留的数据

后续数据库升级建议增加：

- `profit_snapshots`：月度利润测算快照；
- `dividend_plans`：分红测算和老板确认；
- `dividend_payments`：股东实际分红付款；
- `shareholders`：股东信息和持股比例；
- `shareholder_capital_ledger`：股东投入、借款、偿还、分红流水。

首期也可以先不新增表，用 `cash_ledger.shareholder_fund_type` 记录股东资金性质，并在报表中形成只读测算。

## 5. 权限

- 普通员工：不可见利润、现金余额、股东资金和分红；
- 店长：是否可见由老板配置，默认不可见股东分红；
- 财务：可见财务分析和导出，不能独立决定分红；
- 老板：可见全部财务数据，可发起和确认分红方案；
- 后台管理员：仅技术运维权限，不默认查看股东分红金额。

## 6. 上线前确认

上线前必须确认：

1. 股东名单、持股比例和分红规则；
2. 是否按月、季度或不定期分红；
3. 分红前最低现金安全线；
4. 未履约权益风险预留比例；
5. 税费预留比例；
6. 股东投入和股东借款的偿还优先级；
7. 法定利润、税务利润和内部经营利润的差异说明。
