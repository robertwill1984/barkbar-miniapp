# 收入确认与现金流规则 V1.1

日期：2026-07-09｜生效目标：系统 V1.0  
性质：内部经营管理口径；法定会计、税务口径待专业书面确认。

## 1. 六类事实

实际收款、内部经营收入、尚未履约权益、退款、运营支出、现金余额必须分别记录。内部经营收入不得标注为法定会计收入或应税收入。

## 2. 收入确认

### 单次票

支付成功：记实际收款和现金流入，形成未履约余额。实际入园完成：确认内部经营收入并减少未履约余额。

### 10次卡

`单次确认金额 = 实际支付金额 ÷（标准次数 + 赠送次数）`

核销时确认；最后一次处理分币尾差。

### 季卡、年卡

内部经营口径采用逐日摊销：

`每日内部经营收入 = 实际支付金额 ÷ 有效期总自然日`

- 每日生成可追溯确认记录，每月按当月有效天数汇总。
- 最后一个摊销日吸收尾差。
- 暂停期间停止摊销，恢复后顺延。
- 退款时冲减未摊销余额；若需冲减已确认部分，生成反向记录。
- 每张售出卡保存政策和公式快照，有效期内不得更换。

到期审核作为权益结项辅助口径：到期→待处理→检查退款/延期/争议→结项，不代替逐日经营分析。

### 商品与赠品

- 付费商品交付后确认收入，并按移动加权平均成本结转。
- 零元赠饮/礼品不确认销售收入；实际成本分别记会员履约成本或营销/获客成本。

## 3. 现金账户

账户类型：

1. 微信支付商户账户；
2. 银行账户；
3. 门店备用现金；
4. 其他经营收款账户；
5. 押金账户；
6. 受限资金账户；
7. 股东借款账户；
8. 股东投入账户。

期初余额上线前根据平台、银行和盘点录入并由老板确认。

`资金总额 = 全部账户余额合计`

`可动用现金 = 可提现微信余额 + 可自由使用银行余额 + 门店实际备用现金`

押金、保证金、冻结/受限资金、专款专用资金和未明确可用股东款不计可动用现金。股东投入、借款、垫付和偿还分开记录，均不计营业收入。

## 4. 支出审批

状态：`草稿 → 待审批 → 已批准/已驳回 → 待付款 → 已付款 → 已完成`。

- 普通员工可提交单笔≤500元，无审批权。
- 店长可提交≤3000元，可审批其他员工≤500元日常支出。
- >500元原则由老板审批；>3000元必须老板审批。
- 财务复核凭证并登记付款，不单独拥有业务最终批准权。
- 禁止同人申请与最终审批。
- 金额、收款方、用途或凭证实质变化后重新审批。
- 已付款仅能冲正。

有期限、金额上限的老板书面授权须独立记录。

## 5. 库存成本

- 禁止负库存。
- 采用移动加权平均法：`新平均成本 =（原库存成本 + 本次入库成本）÷ 入库后数量`。
- 销售、赠送、报损、领用按当时平均成本出库；历史订单成本锁定。
- 退款回库按原订单结转成本恢复。

## 6. 现金安全与指标

`现金覆盖月数 = 可动用现金 ÷ 最近3个月平均固定现金支出`

- <2月：红色；
- 2-3月：橙色；
- 3-6月：黄色；
- ≥6月：正常。

固定支出至少包括房租、固定工资、基础水电、必要系统费、保险和其他每月必发费用。阈值和清单由老板后台配置并保存生效日期。

经营指标默认：环比降>20%黄、>30%红、连续两月下降提示；沉睡30天、深度沉睡60天、高频30天4次、超高频8次、次卡30天未首核销预警、未来30天到期。全部后台可配置且版本化。

## 7. 月结

- 每月最后一天截止；
- 次月5日前补录；
- 次月6日关账；
- 关账后以调整/冲正处理；
- 历史账期仅老板可批准重开，记录原因、修改和再次关账。

## 8. 备份与导出

- RPO 24小时、RTO 4小时；交易和财务事实实时落库。
- 每日自动备份、每周完整备份、异地保存、每月恢复测试。
- 支持Excel、CSV和PDF汇总；首期不直接对接会计软件。

## 9. 对账

- 支付渠道成功额 = 系统成功支付；
- 账户期末 = 期初 + 流入 - 流出；
- 卡项净收款 = 累计内部经营收入 + 未履约余额 + 退款分摊净额；
- 库存快照 = 全部已入账库存流水之和；
- 所有差异形成异常待办，禁止改原始流水。

## 10. 上线前专业复核

- 会计/税务：法定收入、税点、可抵扣税、平台手续费、凭证期限和导出科目映射。
- 法律：退款扣款、协议提示、个人信息与保存期限。
- 老板：各账户期初、固定支出清单、指标阈值和授权名单。
