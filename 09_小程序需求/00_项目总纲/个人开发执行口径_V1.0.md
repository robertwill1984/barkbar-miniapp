# Bark & Bar 微信小程序｜个人开发执行口径 V1.0

日期：2026-07-10

## 1. 当前执行方式

本项目当前由孙博作为唯一业务负责人和验收人推进，不再默认存在独立产品、设计、开发、测试、法务、会计评审团队。

因此后续文档中的“团队评审”“实施团队评审”统一按以下方式理解：

- 业务口径：孙博确认；
- 产品和页面：Codex 先按已确认资料实现，孙博验收；
- 技术实现：Codex 先按本地可运行、可维护、可继续迭代的方式推进；
- 法律、会计、税务：保留“专业复核建议”，但不阻塞非生产环境开发；
- 微信支付、上线发布：缺少 AppID、商户号、证书、域名时，先做模拟接口和占位配置。

## 2. 是否可以继续开发

可以继续开发。

可立即推进的内容：

- 小程序前端静态页面和交互骨架；
- 使用 Bark&Bar 现有 UI/VI 视觉规范；
- 本地模拟数据；
- 登录、宠物档案、协议、预约、会员页的前端流程；
- 后端模块结构、数据库迁移脚本整理、接口占位；
- 管理后台基础页面。

暂不阻塞开发但生产前必须补齐：

- 微信小程序 AppID；
- 微信支付商户号和 API v3 证书；
- 生产域名和备案；
- 腾讯云生产数据库、对象存储和备份；
- 正式协议文本的最终确认；
- 期初库存、现金账户、员工账号。

## 5. 开发顺序

优先级按个人项目实际可推进程度调整：

1. 小程序前端骨架和 Bark&Bar 视觉样式；
2. 首页、预约、协议、宠物档案、会员中心；
3. 本地 mock 数据和接口封装；
4. 后端基础工程和数据库迁移；
5. 微信登录接入；
6. 支付、动态码核销、退款、库存和财务；
7. 管理后台；
8. 上线配置、备份、监控和试运行。
