# Bark & Bar 微信小程序｜项目开发阶段计划 V1.0

版本：V1.0｜日期：2026-07-09｜排期：待个人开发节奏和微信资质确认

## 1. 阶段与出口

### 阶段 0：业务确认

成果：本轮 10 份 V1.0 文档、规则决策记录。  
出口：待确认问题中所有 P0 项有明确结论，运营手册与线上协议差异已处理。

### 阶段 1：产品与原型

成果：信息架构、低保真原型、状态机、权限矩阵、指标字典。  
出口：顾客预约到入园、购卡到核销、支付退款、商品库存、财务六账流程评审通过。

### 阶段 2：技术详细设计与基础工程

成果：DDL V1、OpenAPI、代码规范、环境、CI、日志、备份方案。  
出口：测试/预发环境可部署，微信登录和权限骨架通过。

### 阶段 3：登录、档案与权限

范围：微信登录、手机号、宠物、协议、员工、RBAC、审计。  
出口：越权测试和敏感字段脱敏通过。

### 阶段 4：预约与入园

范围：容量、预约、审核、签到、健康安全核验、入园记录。  
出口：并发容量、取消/爽约、禁入/管控测试通过。

### 阶段 5：票卡、支付与核销

范围：卡模板/快照、微信支付、动态码、核销、冲正、待兑付和收入确认。  
出口：重复回调/重放/并发核销、每日上限、财务勾稽测试通过。

### 阶段 6：商城、订单与库存

范围：商品、规格、点单、交付、售后、采购、库存流水、成本。  
出口：支付交付、超卖、退款、盘点和成本测试通过。

### 阶段 7：支出、现金和经营看板

范围：支出审批、现金账户、对账、六账报表、老板看板、下钻。  
出口：月度模拟数据对账无差异，指标公式有来源。

### 阶段 8：活动与经营分析

范围：优惠券、老带新、活动 ROI、到期/沉睡/库存风险提示。  
出口：奖励撤销、预算和建议依据测试通过。

### 阶段 9：上线准备

范围：全链路测试、安全、性能、备份恢复、数据初始化、培训、应急预案。  
出口：UAT 签字、回滚方案、门店演练和生产检查表完成。

## 2. 推荐迭代顺序

- MVP-A：登录、宠物、协议、预约、入园核验。
- MVP-B：票卡、微信支付、动态码核销、财务基础流水。
- MVP-C：商城、库存、支出、老板看板。
- 增强版：活动、老带新、经营建议和复杂报表。

## 3. 每阶段质量门

- 需求：规则有来源，待确认项不被静默实现。
- 设计：状态、异常、权限和数据来源完整。
- 开发：代码评审、数据库迁移、幂等和审计。
- 测试：单元、集成、端到端、权限、安全和对账。
- 发布：备份、灰度、监控、回滚、操作手册。

## 4. 必测场景

- 微信回调重复、先回调后前端查询、金额不符。
- 同一动态码重放、截图、两员工同时核销、最后一次并发。
- 10次卡每日第4次、季/年卡当日第2次、非本人到场。
- 预约并发抢最后容量、临时闭园、19:30 后申请。
- 退款跨待兑付与已确认收入、退款后老带新奖励回收。
- 库存最后一件并发、交付重试、盘点冲正。
- 月结后补录、冲正、角色越权、字段脱敏。

## 5. 上线依赖

- 微信小程序主体、商户号、支付 API v3、域名备案。
- 门店规则 P0 决策、隐私政策和协议法律复核。
- 腾讯云生产资源、备份与监控。
- 商品、库存、会员初始数据及现金账户期初余额。
- 员工账号、权限、培训和应急联系人。

## 6. 变更控制

- 重要规则用决策记录编号，并更新 CHANGELOG。
- 已售卡规则不追溯修改；新规则发布新版本。
- 每阶段输出版本化，不覆盖已评审文档。
- 范围新增必须说明业务价值、数据影响、测试成本和上线风险。
