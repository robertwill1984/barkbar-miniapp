# Bark & Bar 微信小程序｜上线配置与资料自检表 V1.0

日期：2026-07-13
适用阶段：本地 MVP、测试服务器、正式上线前

## 1. 当前结论

微信 AppID、AppSecret、商户号和正式服务器资料现在不是阻塞项。

在资料未齐前，当前小程序继续按以下方式推进：

- 本地后端使用 SQLite；
- 微信登录使用本地占位用户；
- 支付使用 `LOCAL_MOCK` 本地模拟支付；
- 退款只记录申请、审核和状态查询，不触发真实微信退款；
- 员工和老板权限暂用本地 PIN 与 `staff_token` 过渡；
- 本地数据库、真实密钥、证书和测试数据不上传 GitHub。

## 2. 资料分阶段提供

| 阶段 | 需要资料 | 是否现在必须 | 用途 |
| --- | --- | --- | --- |
| 本地 MVP | 无 | 否 | 继续完成预约、会员、核销、退款、经营数据演示 |
| 微信登录联调 | 小程序 AppID、AppSecret | 否 | 替换本地占位登录，获取 openid/unionid |
| 正式服务器联调 | API 域名、HTTPS 证书、服务器、数据库 | 否 | 小程序真机访问后端 |
| 微信支付联调 | 微信支付商户号、API v3 Key、商户证书、支付回调域名 | 否 | 替换本地模拟支付 |
| 微信退款联调 | 商户退款权限、退款证书/密钥配置、退款回调域名 | 否 | 替换本地退款审核状态 |
| 正式发布 | 隐私政策、服务协议、类目、备案、体验版测试账号 | 否 | 提交微信审核 |

## 4. 前端配置清单

| 配置 | 当前值 | 正式上线需要 |
| --- | --- | --- |
| `apiBaseUrl` | `http://127.0.0.1:3000/api/v1` | 正式 HTTPS API 域名 |
| `apiEnabled` | `true` | 正式保持 `true` |
| 微信开发者工具 | 勾选“不校验合法域名” | 正式必须配置合法域名 |
| 体验版 | 本地可不提交 | 需要配置体验成员和测试账号 |

正式上线时，小程序后台需要配置：

- request 合法域名；
- uploadFile/downloadFile 合法域名；
- 业务域名；
- 隐私保护指引；
- 服务类目；
- 体验成员；
- 提审版本说明。

## 5. GitHub 上传规则

允许上传：

- 小程序前端代码；
- 后端源代码；
- OpenAPI 文档；
- 部署说明；
- `.env.example` 这种占位示例；
- 不含真实密钥的配置说明。

禁止上传：

- `.env`；
- SQLite 本地数据库；
- 微信 AppSecret；
- 微信支付 API v3 Key；
- 商户证书和私钥；
- 服务器数据库密码；
- COS 密钥；
- 真实用户资料、订单流水、支付流水、退款流水；
- 与小程序无关的施工、合同、预算资料。

## 6. 接正式资料后的执行顺序

1. 先接小程序 AppID/AppSecret，替换本地占位登录；
2. 再接测试服务器 HTTPS API 域名；
3. 再接 PostgreSQL 测试库；
4. 再接微信支付下单和支付回调；
5. 再接微信退款申请、退款查询和退款回调；
6. 最后做体验版真机测试、隐私政策检查和微信审核提交。

## 7. 本地 MVP 继续开发不等待项

以下工作不需要微信资料，可以继续推进：

- 预约、到店核验、会员核销的本地闭环；
- 会员购买、本地支付状态、本地退款状态；
- 员工端查询、审核和权限边界；
- 老板经营数据页；
- OpenAPI、数据库迁移和部署文档；
- 体验版上线前的测试用例和验收清单。
