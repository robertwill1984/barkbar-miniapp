# Bark & Bar 微信小程序｜非程序员上线路线图 V1.0

日期：2026-07-10｜对象：孙博本人

## 1. 先给结论

你不是程序员也可以推进上线，但要把事情分成两类：

1. Codex 可以替你做的：写代码、整理文档、检查配置、生成测试清单、帮你判断错误。
2. 你必须亲自准备或确认的：微信主体、商户号、证书、域名、营业/公司资料、正式协议、真实员工账号、门店初始数据。

开发可以继续，不需要等这些资料全部齐。但正式上线前必须补齐。

## 2. 你最终会经历的 8 个阶段

### 阶段 1：本地演示版

目标：先在电脑里跑起来，看流程对不对。

你要做：

- 安装微信开发者工具；
- 用“游客 AppID / 测试号”打开小程序项目；
- 按页面点一遍：首页、宠物档案、预约、协议、会员、员工核验；
- 把看不懂、不顺、不符合门店实际的地方告诉 Codex。

Codex 做：

- 继续补页面；
- 做本地 mock 数据；
- 修正 UI 和文案；
- 保证你能看懂流程。

### 阶段 2：后端本地版

目标：让小程序不只存在手机页面，还能真正保存数据。

你要做：

- 不需要写代码；
- 只确认业务字段是否够用，比如宠物档案、预约、会员卡、退款、支出。

Codex 做：

- 搭建后端；
- 接数据库；
- 接口和小程序联通；
- 建立测试数据。

### 阶段 3：微信资料准备

目标：拿到上线必须的微信材料。

你要准备：

- 微信小程序账号；
- AppID；
- 小程序管理员微信；
- 认证主体资料；
- 微信支付商户号；
- 商户 API v3 Key；
- 商户证书；
- 支付回调域名。

如果这些还没准备好，开发可以继续，但支付只能先做模拟。

### 阶段 4：测试环境

目标：在一个接近真实的环境里试运行，但不影响真实顾客。

你要做：

- 给 Codex 提供测试用的配置；
- 建几个员工账号；
- 准备测试商品、票卡价格、库存、容量。

Codex 做：

- 部署测试后端；
- 导入数据库；
- 接小程序测试版；
- 建测试账号和权限。

### 阶段 5：门店内部试用

目标：员工先用，发现真实运营问题。

你要安排：

- 店长试预约审核；
- 员工试扫码/核验；
- 财务或你自己试退款、支出、现金流水；
- 每天记录问题。

红线：

- 支付、核销、退款、库存、现金流水有任何重复或错账，不进入正式上线。

### 阶段 6：正式上线前检查

你必须逐项确认：

- 协议文本最终版；
- 隐私政策最终版；
- 票卡价格；
- 预约容量；
- 员工权限；
- 支付配置；
- 备份配置；
- 客服和投诉处理方式；
- 门店现场员工培训。

### 阶段 7：小范围上线

目标：不要一上来放开所有顾客。

建议：

- 先让熟客和会员试用；
- 每天限制预约量；
- 前 7 天每天闭店核对：收款、退款、核销、库存、现金、预约。

### 阶段 8：正式运营

稳定后再开放：

- 全量预约；
- 会员卡购买；
- 活动报名；
- 老带新；
- 商城/吧台；
- 老板经营看板。

## 3. 你每一步只需要问 Codex 什么

不要问“怎么开发”。直接问这些：

- “继续做小程序本地演示版。”
- “帮我检查微信开发者工具为什么打不开。”
- “帮我整理上线前我还缺哪些资料。”
- “帮我生成测试账号和测试数据。”
- “帮我检查这次预约/支付/核销流程有没有问题。”
- “帮我准备提交微信审核的说明。”

## 4. 你现在下一步要做什么

现在最实际的一步：

1. 安装微信开发者工具；
2. 打开目录：`06_小程序前端/`；
3. 如果提示 AppID，先用测试号或游客模式；
4. 看首页、预约、宠物档案、协议、会员中心、员工核验是否符合直觉；
5. 把不顺的地方告诉 Codex。

## 5. 生产上线前不能省的事项

- 不能没有正式隐私政策就收集手机号、宠物资料、疫苗资料。
- 不能没有微信支付商户号就做真实支付。
- 不能没有备份就上线真实财务/会员数据。
- 不能让普通员工看到全店财务。
- 不能允许截图核销。
- 不能把内部经营收入当作法定会计收入直接报税。
