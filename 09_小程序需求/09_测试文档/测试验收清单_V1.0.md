# Bark & Bar 微信小程序｜测试验收清单 V1.0

日期：2026-07-10｜适用：系统 V1.0 开发、联调、UAT、试营业

## 1. 验收原则

- 每条核心经营规则必须有测试用例。
- 支付、退款、核销、库存、现金和收入不只测页面，要测数据勾稽。
- 任何“通过”必须能追溯到测试账号、测试时间、订单/卡/预约编号和截图或日志。
- 法律/会计/税务口径未复核的事项，只能验收内部经营系统是否按已确认业务口径记录，不能标记为法定合规验收完成。

## 3. 支付专项

- JSAPI 下单金额和服务端订单金额一致。
- 支付回调先到、前端后查单，订单状态正确。
- 回调重复10次，不重复记账。
- 金额不一致进入异常，不发放权益。
- 订单关闭后又收到成功回调，进入人工复核。
- 微信退款成功、失败、重复通知均幂等。

## 4. 权限专项

| 角色 | 必测限制 |
|---|---|
| 普通员工 | 看不到累计消费、完整退款历史、全店财务、工资、股东资金 |
| 店长 | 可处理预约和日常会员，但超授权暂停/退款/支出需老板 |
| 财务 | 可对账、付款登记、导出财务范围，但无默认业务终审权 |
| 老板 | 可终审、重开账期、看全量数据 |
| 后台管理员 | 可维护技术配置，但不默认拥有业务财务终审权 |

## 5. 数据勾稽验收

每个测试日结束后抽样核对：

```text
实际收款
- 退款
- 运营支出
+ 股东借款/投入等非收入资金
= 现金账户变化（按账户维度）
```

同时核对：

- 10次卡已核销次数 × 单位确认金额 + 尾差 = 已确认内部收入。
- 季/年卡摊销日历合计 = 截至日期内部收入。
- 待履约权益变化 = 售卡增加 - 核销/摊销/退款/结项减少。
- 商品库存余额 = 期初 + 入库 - 出库 ± 冲正。

## 6. UAT 试运行建议

- 内部模拟：7-14天。
- 门店试运行：14天。
- 每天闭店后输出：
  - 今日预约、到店、爽约；
  - 收款、退款、支出、现金余额；
  - 核销、待履约、内部收入；
  - 库存异常；
  - 权限/审计异常；
  - 员工反馈问题。

## 7. 不通过即停止上线的红线

- 支付、退款或核销出现重复入账且无法自动识别。
- 库存可被扣成负数。
- 普通员工能看到全店财务或敏感信息。
- 新协议发布后无法强制重签。
- 支出/退款可以自批。
- 老板看板数字不能下钻到明细。
- 备份恢复演练失败且无替代方案。
