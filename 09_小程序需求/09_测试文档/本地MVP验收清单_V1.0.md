# Bark & Bar 微信小程序｜本地 MVP 验收清单 V1.0

日期：2026-07-13
适用范围：本地 SQLite + 占位微信登录 + 本地模拟支付 + 本地员工/老板权限

## 1. 验收目标

本清单只判断“本地最小 MVP 是否能完整演示和内部验收”。

以下事项不作为本地 MVP 失败项：

- 真实微信 AppID/AppSecret 未接入；
- 真实微信支付和退款未接入；
- 正式服务器、HTTPS 域名、备案和证书未接入；
- 正式员工微信身份和正式后台未接入；
- 会计、税务、法律专业复核未完成。

本地 MVP 通过后，才进入微信资料、服务器、支付和正式上线联调阶段。

## 3. 本地 MVP 通过标准

本地 MVP 可以标记为通过，需同时满足：

1. 客户能完成预约前核心流程；
2. 员工能处理预约、到店、会员核销和退款审核；
3. 会员购买、购买记录、退款申请和退款状态能闭环；
4. 老板能查看本地经营数据和分红测算依据；
5. 普通员工不能访问老板经营数据；
6. 后端本地构建通过；
7. OpenAPI 文档和本地接口大方向一致；
8. 本地数据库、真实密钥、证书和非小程序资料不进入 GitHub 上传范围。
