# 页面及业务流程清单 V1.0

版本：V1.0｜日期：2026-07-09

## 1. 小程序页面

### 登录与首页

- 启动/授权、手机号绑定、协议确认。
- 首页：今日开放状态、预约入口、购票购卡、点单、会员码、活动。

### 预约

- 日期时段选择、同行人与宠物、票卡选择、健康声明、协议确认。
- 预约支付、预约详情、修改/取消、历史记录。

### 票卡与会员

- 产品列表/详情、确认订单、支付结果。
- 我的卡、卡详情、动态码、使用明细、家庭成员、退款/售后。
- 优惠券、积分（可关闭）、邀请有礼、活动报名。

### 商城

- 分类、商品详情、规格/加料、购物车、结算、订单详情、取餐状态、退款。

### 我的

- 个人资料、宠物档案、联系人、协议记录、预约/消费/售后、消息、隐私与注销。

## 3. 核心流程

### 3.1 预约到入园

```text
登录/建宠物档案
  → 选日期时段/人数/犬只
  → 容量与资格预校验
  → 健康声明/协议
  → 选择票卡
  → 支付或锁定卡资格
  → 待审核/已确认
  → 到店六问+证件健康+现场观察
  → 禁入 / 绿黄红管控
  → 签到
  → 动态码核销/完成票据
  → 入园记录
  → 完成/日报
```

### 3.2 购卡与收入

```text
选择卡项 → 生成规则快照 → 微信支付
→ 实际收款 + 待兑付增加
→ 开卡/激活
→ 次卡核销：待兑付减少 + 收入确认
→ 季/年卡：记录到店，按锁定政策确认
→ 到期：待确认 → 延期/退款/确认收入
```

### 3.3 动态码核销

```text
顾客请求动态码 → 服务端生成短效 nonce
→ 员工扫码 → 校验 nonce/本人/卡状态/日期/每日上限
→ 确认人数犬数与扣次
→ 同一事务：nonce作废 + 扣权益 + 写核销 + 写收入 + 写日志
→ 返回成功凭证
```

失败原因必须明确：码过期、码已用、非本人、余额不足、当日超限、卡冻结/过期、预约不匹配、系统异常。

### 3.4 商品订单

```text
选商品 → 库存预检 → 下单支付 → 收款
→ 接单/制作/备货 → 交付
→ 确认商品收入 + 销售出库 + 结转成本
→ 售后/退款（如有）
```

### 3.5 支出

```text
录入草稿+凭证 → 提交 → 按金额审批
→ 付款 → 现金流出
→ 月结归属 → 报表
→ 错误时冲正，不改原记录
```

### 3.6 退款

```text
申请 → 计算可退/已履约/待兑付/手续费
→ 审批 → 微信退款 → 成功回调
→ 现金减少 + 待兑付/收入冲减
→ 权益回收 + 奖励撤销 + 日志
```

## 4. 状态流转

- 预约：草稿、待支付、待审核、已确认、已签到、已完成、已取消、已拒绝、爽约、已过期。
- 订单：待支付、已支付、处理中、待交付、已完成、已取消、退款中、部分退款、已退款。
- 卡：待激活、有效、暂停、冻结、到期待确认、到期、退款中、已退款、作废。
- 支付：初始化、处理中、成功、失败、关闭。
- 退款：申请、审核中、已拒绝、处理中、成功、失败、已关闭。
- 库存流水：待入账、已入账、已冲正。
- 收入确认：已入账、已冲正。

## 5. 页面间关键跳转

- 预约详情 → 支付/动态码/入园记录/取消。
- 会员卡详情 → 动态码/核销明细/退款申请。
- 后台预约详情 → 入园核验 → 核销 → 入园记录。
- 订单详情 → 支付/交付/库存流水/退款。
- 看板指标 → 日汇总 → 业务明细 → 原订单/支付/核销。
- 卡项报表 → 用户卡 → 收入确认/待兑付/退款。

## 6. 异常与人工兜底

- 弱网：不允许离线核销；可登记“待系统恢复”，恢复后由店长处理，禁止口头销账。
- 支付回调延迟：主动查单，不让顾客重复支付。
- 动态码失败：刷新码；不得仅凭手机号、截图或熟人关系放行。
- 争议退款/到期：冻结待处理，老板/财务审批。
- 禁入犬只：记录原因并终止入园；红色管控不等于禁入。

## 7. 运营手册联动

- 预约与核验页必须显示入园六问和禁入规则。
- 泳池页必须显示 0.8-1m、无浅水区、小狗或首次游泳须穿游泳衣。
- 员工工作台提示 15-20 分钟巡场和 19:30 最晚入园。
- 会员权益不能跳过安全核验与协议。
