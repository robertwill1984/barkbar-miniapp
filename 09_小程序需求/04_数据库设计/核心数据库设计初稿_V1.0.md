# Bark & Bar 微信小程序｜核心数据库设计初稿 V1.0

版本：V1.0｜日期：2026-07-09｜数据库：PostgreSQL 16+

## 1. 设计约定

- 主键统一 `id uuid PK DEFAULT gen_random_uuid()`。
- 金额统一 `bigint`，单位为分，字段后缀 `_amount`。
- 时间统一 `timestamptz`，服务端存 UTC，展示按 Asia/Shanghai。
- 所有业务表含 `created_at timestamptz NOT NULL DEFAULT now()`、`updated_at timestamptz NOT NULL DEFAULT now()`。
- 可编辑主数据可含 `deleted_at` 软删除；支付、退款、核销、收入、待兑付、现金、库存和审计流水禁止删除。
- 状态使用受控 `varchar` + CHECK 或枚举；规则快照用 `jsonb`，但关键查询字段必须独立列。
- 外键默认 `RESTRICT`；历史记录不得因主数据删除而级联消失。
- 敏感字段应用层加密，检索需要独立盲索引/哈希。

字段简写：`NN`=必填，`D`=默认值，`UQ`=唯一，`FK`=外键。

## 2. 身份、权限与档案

### 2.1 `users`

用途：顾客主档。

字段：

- `wechat_openid varchar(128) NN`、`wechat_unionid varchar(128)`：微信身份。
- `phone_ciphertext text`、`phone_hash char(64)`：加密手机号与检索哈希。
- `nickname varchar(80)`、`avatar_file_id uuid FK files`。
- `status varchar(20) NN D 'ACTIVE'`：`ACTIVE/FROZEN/CLOSED`。
- `last_login_at timestamptz`。

约束/索引：`UQ(app_id,wechat_openid)`（实现时增加 `app_id`）；`UQ phone_hash WHERE phone_hash IS NOT NULL`；索引 `status,created_at`。  
关系：1:N pets、reservations、orders、user_cards。  
状态：ACTIVE↔FROZEN→CLOSED。

### 2.2 `pets`

- `owner_user_id uuid NN FK users`
- `name varchar(80) NN`、`breed varchar(100) NN`、`sex varchar(10)`、`birth_date date`、`weight_kg numeric(6,2)`
- `vaccination_status varchar(20) NN D 'UNKNOWN'`、`vaccination_expires_on date`、`neutered boolean`
- `behavior_level varchar(20) NN D 'UNASSESSED'`：`GREEN/YELLOW/RED_CONTROL/DENIED/UNASSESSED`
- `attack_history boolean NN D false`、`health_notes text`、`behavior_notes text`
- `photo_file_id uuid FK files`、`status varchar(20) NN D 'ACTIVE'`

索引：`owner_user_id,status`、`vaccination_expires_on`、`behavior_level`。  
关系：N:M reservations/admissions 通过明细表。  
状态：ACTIVE/FROZEN/DECEASED/ARCHIVED；风险等级变更写审计。

### 2.3 `admins`

- `username varchar(80) NN UQ`、`password_hash text NN`
- `display_name varchar(80) NN`、`phone_ciphertext text`、`phone_hash char(64)`
- `status varchar(20) NN D 'ACTIVE'`、`last_login_at timestamptz`

索引：`status`；关系 N:M roles。状态 ACTIVE/FROZEN/LEFT。

### 2.4 `roles`、`permissions`、`admin_roles`、`role_permissions`

- `roles`: `code varchar(50) NN UQ`, `name varchar(80) NN`, `data_scope varchar(20) NN D 'STORE'`, `status`.
- `permissions`: `code varchar(100) NN UQ`, `name`, `resource`, `action`, `sensitive boolean NN D false`.
- `admin_roles`: `admin_id FK`, `role_id FK`, 联合 PK。
- `role_permissions`: `role_id FK`, `permission_id FK`, 联合 PK。

关系：管理员—角色—权限多对多。角色停用不删除历史授权记录。

### 2.5 `agreements`、`agreement_acceptances`

- `agreements`: `type`, `version`, `title`, `content_hash`, `content_file_id FK`, `effective_at`, `status`; `UQ(type,version)`.
- `agreement_acceptances`: `agreement_id FK`, `user_id FK`, `pet_id FK NULL`, `accepted_at NN`, `ip`, `device_info jsonb`, `evidence_hash`; `UQ(agreement_id,user_id,pet_id)`（NULL 需部分唯一索引）。

状态：DRAFT→PUBLISHED→RETIRED；已发布内容不可覆盖。

## 3. 预约与入园

### 3.1 `reservation_slots`

- `service_date date NN`、`start_time time NN`、`end_time time NN`
- `people_capacity int NN`、`pet_capacity int NN`
- `booked_people int NN D 0`、`booked_pets int NN D 0`
- `status varchar(20) NN D 'OPEN'`：OPEN/CLOSED/FULL
- `rule_snapshot jsonb NN`

约束：`UQ(service_date,start_time,end_time)`；CHECK 容量与已订数量非负且已订不超过容量。索引 `service_date,status`。

### 3.2 `reservations`

- `reservation_no varchar(32) NN UQ`
- `user_id uuid NN FK users`、`slot_id uuid NN FK reservation_slots`
- `people_count int NN`、`pet_count int NN`
- `usage_type varchar(20) NN`：TICKET/USER_CARD/ONSITE
- `user_card_id uuid FK user_cards`、`order_id uuid FK orders`
- `status varchar(30) NN D 'DRAFT'`
- `reviewed_by uuid FK admins`、`reviewed_at`、`cancelled_at`、`cancel_reason`
- `version int NN D 1`、`health_declaration jsonb NN`

约束：人数/犬数 >0；卡方式必须有 user_card_id。索引 `slot_id,status`、`user_id,created_at`、`service_date` 通过 slot 或冗余列。  
状态：DRAFT→PENDING_PAYMENT/PENDING_REVIEW→CONFIRMED→CHECKED_IN→COMPLETED；旁路 CANCELLED/REJECTED/NO_SHOW/EXPIRED。

### 3.3 `reservation_pets`

`reservation_id FK`、`pet_id FK`、`is_swimming boolean D false`、`life_jacket_required boolean D false`；联合 PK。用于锁定本次宠物与泳池要求。

### 3.4 `admissions`

- `admission_no varchar(32) NN UQ`
- `reservation_id uuid FK reservations`、`user_id uuid NN FK users`
- `arrived_at timestamptz NN`、`left_at timestamptz`
- `people_count int NN`、`pet_count int NN`
- `risk_result varchar(20) NN`、`checklist_snapshot jsonb NN`
- `operator_id uuid NN FK admins`、`status varchar(20) NN D 'IN_PROGRESS'`

关系：1:N admission_pets、redemptions；索引 `arrived_at,status`。状态 IN_PROGRESS→COMPLETED；或 DENIED/CANCELLED。

### 3.5 `admission_pets`

`admission_id FK`、`pet_id FK`、`risk_level NN`、`swimming boolean D false`、`life_jacket_confirmed boolean`、`notes`；联合 PK。

## 4. 产品、卡项与权益

### 4.1 `card_products`

- `code varchar(50) NN UQ`、`name varchar(100) NN`
- `card_kind varchar(20) NN`：PUNCH/SEASON/ANNUAL/CUSTOM
- `list_price_amount bigint NN`、`sale_price_amount bigint NN`
- `validity_value int NN`、`validity_unit varchar(10) NN`
- `paid_units int`、`bonus_units int NN D 0`
- `included_people int NN D 2`、`included_pets int NN D 1`
- `daily_use_limit int`、`family_member_limit int NN D 0`
- `transferable boolean NN D false`、`pausable boolean NN D false`、`refundable boolean NN D true`
- `recognition_policy varchar(30) NN`：PER_REDEMPTION/AT_EXPIRY/STRAIGHT_LINE
- `rule_version int NN D 1`、`rules jsonb NN`、`status varchar(20) NN D 'DRAFT'`

约束：金额≥0、有效期>0；发布后不可原地修改，复制新版本。索引 `status,card_kind`。状态 DRAFT→ON_SALE→OFF_SALE→RETIRED。

### 4.2 `user_cards`

- `card_no varchar(32) NN UQ`、`user_id FK users`、`card_product_id FK card_products`
- `order_item_id uuid NN FK order_items`
- `product_snapshot jsonb NN`、`recognition_policy NN`
- `paid_amount bigint NN`、`paid_units int`、`bonus_units int NN D 0`
- `remaining_units int`、`deferred_balance_amount bigint NN`
- `activated_at`、`valid_from`、`valid_until`、`status NN`
- `lock_version int NN D 0`

索引：`user_id,status`、`valid_until,status`；CHECK 余额和次数非负。  
状态：PENDING_ACTIVATION→ACTIVE↔PAUSED/FROZEN→EXPIRED_PENDING_REVIEW→EXPIRED/CLOSED；退款旁路。

### 4.3 `card_family_members`

`user_card_id FK`、`name`、`phone_ciphertext`、`phone_hash`、`relationship`、`status`；`UQ(user_card_id,phone_hash)`。数量由卡快照限制。

### 4.4 `card_benefits`、`user_card_benefits`

- 模板：`card_product_id FK`, `benefit_type`, `quantity`, `reset_period`, `rules jsonb`.
- 实例：`user_card_id FK`, `benefit_type`, `granted_qty`, `used_qty D 0`, `valid_until`, `status`.

用于赠饮、礼品、券等非主卡权益；使用必须产生权益流水。

### 4.5 `redemptions`

- `redemption_no varchar(32) NN UQ`、`batch_no varchar(32) NN`
- `user_card_id uuid NN FK user_cards`、`admission_id uuid FK admissions`、`reservation_id uuid FK reservations`
- `units int NN`、`recognized_amount bigint NN D 0`
- `nonce_hash char(64)`、`idempotency_key varchar(100) NN UQ`
- `operator_id uuid NN FK admins`、`redeemed_at timestamptz NN`
- `status varchar(20) NN`、`failure_code varchar(50)`、`reversed_redemption_id uuid FK redemptions`

约束：`UQ(nonce_hash) WHERE nonce_hash IS NOT NULL`；units>0。索引 `user_card_id,redeemed_at`、`operator_id,redeemed_at`。状态 PENDING→SUCCESS→REVERSED 或 FAILED。

## 5. 商品、订单、支付与退款

### 5.1 `product_categories`、`products`、`product_skus`

- 分类：`name`, `sort_order D 0`, `status`.
- 商品：`category_id FK`, `name`, `type`, `description`, `cover_file_id FK`, `status`.
- SKU：`product_id FK`, `sku_code UQ`, `specs jsonb`, `price_amount`, `member_price_amount`, `cost_method D 'MOVING_AVERAGE'`, `track_inventory boolean`, `status`.

索引：分类/上下架状态；SKU code 唯一。状态 DRAFT/ON_SALE/OFF_SALE/RETIRED。

### 5.2 `orders`

- `order_no varchar(32) NN UQ`、`user_id FK users`
- `order_type varchar(20) NN`：TICKET/CARD/RETAIL/MIXED
- `gross_amount`、`discount_amount D 0`、`payable_amount`、`paid_amount D 0`、`refunded_amount D 0`
- `fulfillment_type`、`status NN`、`paid_at`、`completed_at`、`cancelled_at`
- `idempotency_key varchar(100) NN UQ`

CHECK 金额非负且折扣≤毛额。索引 `user_id,created_at`、`status,created_at`。

### 5.3 `order_items`

- `order_id FK`、`item_type NN`、`product_ref_id uuid NN`
- `name_snapshot`、`rule_snapshot jsonb NN`
- `quantity int NN`、`unit_price_amount`、`discount_amount D 0`、`net_amount`
- `fulfilled_qty int NN D 0`、`status NN`

索引 `order_id`、`item_type,product_ref_id`。一项可关联票、卡或 SKU；应用层加约束。

### 5.4 `payments`

- `payment_no UQ`、`order_id FK`、`channel NN`
- `amount NN`、`channel_transaction_no varchar(100)`、`status NN`
- `requested_at`、`succeeded_at`、`callback_payload_hash`
- `idempotency_key UQ`

约束：`UQ(channel,channel_transaction_no) WHERE ...`。状态 INIT→PROCESSING→SUCCESS/FAILED/CLOSED。成功记录不可修改。

### 5.5 `refunds`

- `refund_no UQ`、`order_id FK`、`payment_id FK`
- `requested_amount NN`、`approved_amount`、`succeeded_amount D 0`
- `reason NN`、`calculation_snapshot jsonb NN`
- `requested_by`、`approved_by FK admins`、`channel_refund_no`
- `status NN`、`succeeded_at`、`idempotency_key UQ`

状态 REQUESTED→REVIEWING→PROCESSING→SUCCESS；或 REJECTED/FAILED/CLOSED。索引订单、支付、状态。

## 6. 库存与采购

### 6.1 `inventory_balances`

`sku_id uuid PK FK product_skus`、`on_hand_qty numeric(14,3) NN D 0`、`reserved_qty D 0`、`average_cost_amount bigint D 0`、`lock_version D 0`。  
这是性能快照；真实依据为流水。

### 6.2 `inventory_movements`

- `movement_no UQ`、`sku_id FK`、`movement_type NN`
- `quantity_delta numeric(14,3) NN`、`unit_cost_amount bigint`
- `source_type`、`source_id uuid`、`occurred_at NN`
- `operator_id FK admins`、`status NN`、`reversal_of_id FK inventory_movements`

`UQ(source_type,source_id,sku_id,movement_type)` 防重复；状态 POSTED/REVERSED。类型 PURCHASE/SALE/RETURN/USE/DAMAGE/COUNT_ADJUST。

### 6.3 `suppliers`、`purchase_orders`、`purchase_order_items`

- 供应商：`code UQ`, `name`, `contact`, `phone_ciphertext`, `status`.
- 采购单：`purchase_no UQ`, `supplier_id FK`, `total_amount`, `status`, `ordered_at`, `received_at`, `operator_id`.
- 明细：`purchase_order_id FK`, `sku_id FK`, `qty`, `unit_cost_amount`, `received_qty D 0`, `expiry_date`.

采购状态 DRAFT→APPROVED→PARTIALLY_RECEIVED→RECEIVED→CLOSED；可 CANCELLED。

## 7. 活动与优惠

### 7.1 `campaigns`

`name`, `type`, `starts_at`, `ends_at`, `audience_rule jsonb`, `benefit_rule jsonb`, `budget_amount`, `quota`, `stackable boolean`, `status`。  
索引时间和状态；状态 DRAFT→SCHEDULED→ACTIVE→ENDED/CANCELLED。

### 7.2 `coupons`

`coupon_no UQ`, `campaign_id FK`, `user_id FK`, `type`, `face_value_amount`, `threshold_amount`, `valid_from`, `valid_until`, `status`, `used_order_id FK`。  
状态 ISSUED→LOCKED→USED；或 EXPIRED/REVOKED。

### 7.3 `referrals`

`inviter_user_id FK`、`invitee_user_id FK`、`code`、`registered_at`、`qualified_at`、`qualifying_order_id FK`、`status`、`risk_flags jsonb`。  
`UQ(invitee_user_id)`（首客单一邀请关系）；索引 inviter+月份用于每月奖励上限。

## 8. 财务与经营事实

### 8.1 `revenue_recognitions`

- `recognition_no UQ`、`source_type NN`、`source_id uuid NN`
- `user_card_id FK`、`order_item_id FK`
- `amount bigint NN`（冲正为负或通过关联反向记录）
- `recognition_date date NN`、`policy NN`、`calculation_snapshot jsonb NN`
- `status NN D 'POSTED'`、`reversal_of_id FK revenue_recognitions`

`UQ(source_type,source_id,policy)` 防重复；索引日期、卡、订单项。禁止更新/删除。

### 8.2 `deferred_entitlement_ledger`

- `entry_no UQ`、`user_card_id FK`
- `entry_type NN`：SALE/REDEMPTION/REFUND/EXPIRY_RECOGNITION/ADJUSTMENT/REVERSAL
- `amount_delta bigint NN`：增加待兑付为正，减少为负
- `source_type/source_id`、`occurred_at NN`、`balance_after bigint NN`
- `reversal_of_id FK`

约束：`UQ(source_type,source_id,entry_type)`；索引卡+时间。禁止删除。

### 8.3 `cash_accounts`、`cash_ledger`

- 账户：`code UQ`, `name`, `account_type`, `status`, `opening_balance_amount`.
- 流水：`entry_no UQ`, `account_id FK`, `direction INFLOW/OUTFLOW`, `business_type`, `amount NN`, `source_type/source_id`, `value_at NN`, `counterparty`, `status`, `reversal_of_id`.

`UQ(source_type,source_id,account_id,business_type)`；索引账户+日期。内部转账使用同一 transfer_group_id。

### 8.4 `operating_expenses`

- `expense_no UQ`、`category NN`、`amount NN`、`business_date NN`、`belonging_month date NN`
- `payee NN`、`cash_account_id FK`、`payment_method`
- `operator_id FK admins`、`approver_id FK admins`
- `voucher_file_id FK files`、`invoice_status`、`is_fixed boolean D false`
- `status NN`、`paid_at`、`reversal_of_id FK`

索引月份+分类、状态、付款账户。状态 DRAFT→SUBMITTED→APPROVED→PAID→REVERSED。

### 8.5 `financial_adjustments`

`adjustment_no UQ`, `target_type`, `target_id`, `before_snapshot jsonb`, `after_snapshot jsonb`, `reason`, `evidence_file_id`, `requested_by`, `approved_by`, `status`, `posted_at`。  
状态 REQUESTED→APPROVED→POSTED 或 REJECTED；实际调整仍需写对应反向流水。

### 8.6 `metric_snapshots`

`metric_code`, `period_type`, `period_start`, `period_end`, `value_numeric`, `dimension jsonb`, `formula_version`, `source_max_time`, `calculated_at`。  
`UQ(metric_code,period_type,period_start,period_end,dimension_hash,formula_version)`。该表是缓存，可重算，不是财务事实源。

## 9. 文件与审计

### 9.1 `files`

`storage_key UQ`, `bucket`, `mime_type`, `size_bytes`, `sha256`, `owner_type`, `owner_id`, `visibility D 'PRIVATE'`, `status`。  
状态 UPLOADING→ACTIVE→QUARANTINED/DELETED；财务凭证删除需受限。

### 9.2 `audit_logs`

`request_id`, `actor_type`, `actor_id`, `action`, `resource_type`, `resource_id`, `before_data jsonb`, `after_data jsonb`, `reason`, `ip`, `user_agent`, `occurred_at NN D now()`。  
索引资源+时间、操作者+时间、request_id。只追加，单独权限，建议归档后防篡改。

## 10. 一致性与并发

- 预约容量：锁定 `reservation_slots` 行后比较并更新已订数。
- 核销：锁定 `user_cards` 行；nonce、幂等键、核销号三重唯一。
- 支付/退款：渠道交易号和幂等键唯一。
- 库存：锁定余额快照并写流水；库存不足按策略拒绝。
- 财务：业务事件、流水和 outbox 同事务；异步消费者以事件 ID 幂等。
- 报表：只读已入账状态，冲正记录纳入净额。
